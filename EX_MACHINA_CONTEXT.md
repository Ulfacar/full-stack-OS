## Что это

SaaS-платформа (OS) для управления AI-ботами отелей. Мультитенант: один инстанс, много отелей. Клиент-отель регистрируется → настраивает бота → получает webhook для Telegram/WhatsApp.

---

## Бизнес-модель

- **$700 разово** за подключение + **$20/мес** подписка
- AI через OpenRouter — один общий ключ, трекинг токенов по `hotel_id` (таблица `ai_usage`)
- Клиент не видит токены — всё включено в подписку

## Первый клиент — Ton Azure (уроки)

~5 диалогов/день, летом больше. Бектуру (владелец) важны: обращения, брони, канал. Токены/AI метрики — бесполезны.

### Критичные правила

1. **Промпт = 70% запретов.** Min 15 "НЕ делай"
2. **Post-processing обязателен** — AI нарушает промпт 10–20% времени. Pipeline: `is_garbled → needs_operator → clean_response → fix_prices → ensure_variants`
3. `clean_response` вырезает: теги, сезоны, навязчивые вопросы (~40 паттернов), опечатки
4. `is_garbled`: >15 двоеточий или >3 повторов слов подряд → retry
5. **Booking flow**: только когда гость САМ просит. Чеклист: даты + гости + ФИО + телефон → `[НУЖЕН_МЕНЕДЖЕР]`
6. **Стиль бота настраиваемый** — Ton Azure хотел "чётко и по делу", откатили навязчивого
7. **WhatsApp debounce 1.5 сек** — склеивает быстрые сообщения
8. **Цены** — простой текст с пайпами, НЕ markdown таблицы
9. **Temperature 0.3, max_tokens 800** — оптимально
10. Голосовые через Whisper API — топ запрос, в roadmap
11. Антипаттерн "итого за N ночей" — бот считает неправильно, показывать цену/сутки

### Бот НЕ может

- Цены ресторана
- Конкретные номера (этаж / вид)
- Окрестности (банкомат / аптека)
- Сложные калькуляции

### Followup

10 мин + 15 мин, макс 2 сообщения. Жалоб нет. AI → менеджер: без таймаута, авто-закрытие через 4ч.

## WhatsApp через wappi.pro

- Не более 5–10 сообщений/мин, персонализация, иначе бан
- Новый номер прогревать 7–14 дней (входящих > исходящих)
- Телефон НЕ должен быть постоянно включён — бот на сервере wappi.pro
- Устройство нужно только для первичной привязки
