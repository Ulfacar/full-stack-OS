# Perplexity prompt — Sales presentation review v2

> **Назначение:** готовый промпт для повторного анализа sales-презентации через Perplexity. В первой итерации (25.04) Perplexity дал хорошие идеи по структуре, но галлюцинировал цифры (377 отелей в Чолпон-Ате, 2.3M туристов, ROI 300%) и использовал устаревшую цену $500+$15. Этот промпт даёт ему весь контекст чтобы он не мог сбиться.
>
> **Когда использовать:** перед каждым крупным обновлением sales-deck (раз в квартал или при добавлении новых intel-данных).

═══════════════════════════════════════════════════════
SALES-АРСЕНАЛ (использовать как источник)
═══════════════════════════════════════════════════════

У нас уже задокументированы 8 критичных багов HotelBot (после
тестирования их прототипа @hotel_lang_bot 23.04.2026):
1. Нет temporal validation — бот принял заезд 24 мая, выезд 28
   марта (опечатка) и насчитал 308 ночей = счёт 6,481,413 рублей
   без warning
2. Кыргызский = translation layer — на запрос на KG бот возвращает
   перевод СВОЕГО ПРЕДЫДУЩЕГО RU-ответа, не отвечает на новый
   запрос
3. Mix RU+KG = автоматический handoff на оператора (не отвечает по
   сути)
4. RAG-галлюцинация — на вопрос про кровати/коляску ответил про
   СПА-услуги
5. Handoff inconsistency — срабатывает где не надо, не работает
   где надо
6. Random language switch — бот переключает RU/EN/KG без триггера
7. Cross-message memory overflow — переносит контекст из
   предыдущих несвязанных запросов
8. Дублирование сообщений — race condition в их очереди

Также Ольга (sales HotelBot) призналась 24.04 что они НЕ продумали
сценарий mix RU+KG ("передала программистам, жду ответ"). Их
собственные разработчики не знают как бот должен реагировать.
