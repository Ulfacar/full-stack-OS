## Цикл: 1-недельный спринт

**Длительность:** понедельник 09:00 → пятница 20:00 (по часовому поясу Алана).
**Почему 1 неделя:** принцип *Fast start / Fast die* из бизнес-тренинга. Короткий цикл = быстрая обратная связь. Если идея не работает — понимаешь через неделю, не через месяц.
**Когда 2 недели:** только если спринт целиком про миграции / инфру без UI (Sprint 1). Явно договариваемся заранее, не по умолчанию.

## Ceremonies (минимальная версия для solo-dev)

### 1. Sprint Planning — понедельник, 30 минут

**Цель:** выбрать 3-5 карточек из Trello Backlog → перенести в колонку «Sprint N».

**Чеклист:**
- [ ] Открыть Trello «Backlog», отсортировать по приоритету (R1a > R1b > R1c > R2)
- [ ] Выбрать карточки по ёмкости: S = 1-2ч, M = полдня-день, L = 2-3 дня. Total ≤ 5 рабочих дней.
- [ ] Проверить DoR (Definition of Ready) для каждой (см. ниже)
- [ ] Переместить в колонку «Sprint N»
- [ ] Записать в Telegram-заметку «Sprint N цель: …» одной фразой

**Правила скоупа:**
- ≥1 story High-$-impact (инсайт #2 refounding: деньги-first)
- ≤2 story тех-долга на спринт (иначе тонешь в рефакторинге)
- 0 L-story без обсуждения — если L, сначала разбить на 2×M

### 2. Daily Scrum — 5 минут в TG-заметку «Ex-Machina — лог»

**Формат** (копипаст, без воды):

```
[YYYY-MM-DD]
Вчера: #29 fix_prices портирован, тест на edge-case 0-цены passed
Сегодня: #20 реквизиты-поле в визарде + промпт
Блокер: ждёт решение Алана — показывать ли реквизиты в chat-превью клиенту
```

Не пишешь если ничего не сделал — тоже сигнал (фиксировать в ретро).

### 3. Sprint Review — пятница 15:00, 20 минут

**Цель:** проверить что спринт-карточки **реально работают**, не просто закрыты.

**Чеклист:**
- [ ] Каждая story из Sprint N проверена end-to-end (frontend → backend → БД)
- [ ] UI-фичи открыл в браузере (Playwright MCP или руками), проверил golden path + edge case
- [ ] API-фичи проверил через curl или Swagger
- [ ] Миграции применены, откатываются
- [ ] Карточки переведены в «Done» или возвращены в «In Progress» с комментом «не доделано»

Не путать с code review — это приёмка от себя как PO.

### 4. Retrospective — пятница 16:00, 45 минут

**Формат:** `docs/process/retro_template.md` скопировать в `docs/retros/retro_YYYY_MM_DD.md`, заполнить.

45 минут долго для solo — но это **единственный момент обучения** за спринт. Не сокращать.

## Definition of Ready (DoR)

Карточка готова к взятию в спринт когда:

- [ ] Title понятен из одной строки
- [ ] `$ impact` указан (High / Med / Low)
- [ ] `R` (R1a/R1b/R1c/R2/R3) указан
- [ ] `Size` оценка (S/M/L), L — только если есть разбивка на подпункты в AC
- [ ] **Why** написан (почему делаем — 1-2 предложения, со ссылкой на USM story / Lean Canvas)
- [ ] **Acceptance Criteria** (список чекбоксов, verifiable — не «работает хорошо», а «вернул 200 OK при X»)
- [ ] Блокеры из `blocker-графа` разрешены (см. USM blocker-graph)

Если карточка не проходит DoR — **не берём в спринт**, отправляем в Backlog доработать.

---

## Definition of Done (DoD)

Story считается Done когда:

- [ ] Код закоммичен в main (модульные коммиты, формат `feat(area): verb` / `fix(area): verb`)
- [ ] Все AC из карточки отмечены ✅
- [ ] Если миграция — протестирована на локальной БД (upgrade + downgrade)
- [ ] Если UI — открыт в браузере, golden path работает
- [ ] Если API — есть curl-пример в описании карточки Trello или в коммите
- [ ] Railway deploy успешен (для Ex-Machina production) или явно отложен с Why
- [ ] Если new tech debt обнаружен — создана новая карточка в Backlog, не остаётся в голове

**Не считается Done:**
- «Работает локально, потом задеплою» — это In Progress
- «Протестировал частично» — это In Progress
- «Миграция применена но данные руками не перенёс» — это Blocked

## Размеры (S/M/L)

| Size | Время | Пример |
|---|---|---|
| **S** | 1-4 часа | Добавить поле в визард, написать md-гайд, портировать одну функцию |
| **M** | Полдня-день | Endpoint + tests + UI, миграция + backfill script |
| **L** | 2-3 дня | Миграции 014-018, двусторонний TG-канал, ROI-отчёт end-to-end |
| **XL** | >3 дней | ❌ Запрещено. Разбить на 2-3 L или M перед DoR |

**Хитрость:** неизвестные задачи всегда оцениваешь на 1 ступень выше. Никогда не «кажется S» — если не уверен, ставь M.

## Колонки Trello

```
┌─────────┐  ┌──────────┐  ┌─────────┐  ┌────────────┐  ┌────────┐  ┌──────┐  ┌─────────┐
│ Backlog │→ │ Sprint 0 │→ │Sprint N │→ │In Progress │→ │ Review │→ │ Done │  │ Blocked │
└─────────┘  └──────────┘  └─────────┘  └────────────┘  └────────┘  └──────┘  └─────────┘
```

- **Backlog** — всё что когда-то сделаем. Сортировка: R1a → R1b → R1c → R2 → R3
- **Sprint 0** — предспринт (параллельно с Sprint 1), содержит только solo-tasks без блокеров. Сейчас = `fix_prices` порт
- **Sprint N** — карточки выбранные на текущую неделю (Monday planning)
- **In Progress** — что делаешь прямо сейчас. Max 2 карточки одновременно (иначе контекст-свитчинг убивает скорость)
- **Review** — закрыто по коду, но ещё не проверено end-to-end (Friday Review)
- **Done** — приёмка пройдена, merged, deploy
- **Blocked** — ждёт внешнее (миграция сверху, решение от Назиры, ответ HotelBot 😏). Обязательно Why в карточке

## Что НЕ делаем (антипаттерны solo-дёва)

- ❌ **Plot twist в середине спринта.** Если Назира / клиент внезапно просит срочную фичу — кидаем в Backlog с меткой «urgent», обсуждаем в следующем planning. Спринт-скоуп **не меняется внутри недели**.
- ❌ **Эпики.** Если чувствуешь что нужен epic — это просто плохо разбитая L-story. Разбей на S/M.
- ❌ **Velocity как KPI.** Считаем количество done-story, но не стремимся «гнать больше». Цель: стабильный выход 3-5 stories/нед.
- ❌ **Планирование 3 спринтов вперёд.** Только текущий + тема следующего. Всё остальное — в USM / Lean Canvas / Roadmap.
- ❌ **"Потом протестирую".** Если не протестировано в Sprint Review — не Done.
