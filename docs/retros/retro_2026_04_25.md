## 🧠 Инсайты уровня продукта

1. **HotelBot не таргетит mini-hotel КР целенаправленно — подтверждено новым datapoint.** Их собственные программисты не знают как бот должен реагировать на mix RU+KG (Ольга 24.04 «Передала программистам, жду ответ»). Это не процессное наблюдение, а **продуктовое**: наша монопольная ниша усилена. **Перенос:** в `docs/strategy/lean_canvas_v1.md` блок 9 (Unfair Advantage) — добавить «mix RU+KG — необдуманный edge case у конкурентов, у нас встроено».

2. **HotelBot Exely будет 6 недель адаптации с нуля.** Это даёт нам конкретный sales-talking-point: **«у нас Distribution API ранний доступ — готовая интеграция Exely; у HotelBot минимум 6 недель кастомной разработки за тот же ценник $6675»**. **Перенос:** в `docs/sales/hotelbot_comparison_2026_04.md` уже сделано в этой сессии.

3. **HotelBot 308-ночей bug — мощный sales hook.** Реальный кейс «бот выставит счёт на год вместо недели» — аргумент ценности нашего `fix_prices` validator для любого потенциального клиента. **Перенос:** в `docs/sales/hotelbot_comparison_2026_04.md` уже сделано (раздел B1 Critical bugs).

4. **ROI-формула для `#33` калибруется на их Абхазский кейс** (900+ диалогов / 80% автоматизации / +5% броней). Это даёт нам референс-метрики для расчёта `(N подтверждённых × avg$) ÷ $40 = XX× ROI`. **Перенос:** при работе над `#33` (Sprint 3 кандидат) — использовать эти цифры как калибр.

5. **Sales-asset формат работает** — TL;DR за 30 сек + контр-таблица + готовые ответы на возражения. Создавать такой же для NURAI как только закроем их intel-трек. **Перенос:** action item в Backlog.

6. **Custom-pricing у конкурентов = наш Diff.** HotelBot скрывает цены до созвона, NURAI прайс есть на сайте но не публикует hotel-кейсы. У нас **прозрачная цена $800+$40 на лендинге** — это killfear для small-hotel КР. **Перенос:** усилить эту позицию на `#1 Лендинг «Ex-Machina для мини-отелей КР»` (R1c).
